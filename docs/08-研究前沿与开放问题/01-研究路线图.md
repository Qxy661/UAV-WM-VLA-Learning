# 从学习到研究：无人机智能体研究路线图

> 预计阅读：15 分钟 | 前置知识：已完成基础概念学习（Part 1）

---

## 为什么需要这份路线图

学习现有论文是起点，但科研的核心是**发现和解决问题**。本文档帮你从"读懂论文"过渡到"做出贡献"。

```
学习者路径:  读论文 → 理解方法 → 复现实验 → 完成
研究者路径:  读论文 → 批判思考 → 发现问题 → 提出假设 → 设计实验 → 验证 → 贡献
```

---

## 科研流程五步法

### Step 1: 建立领域地图（已完成）

通过本项目的 Part 1-4，你已经建立了：
- VLM、VLA、世界模型的概念框架
- 三大技术族的发展脉络
- 无人机领域的特殊挑战

**检验标准**：能用一张图画出 VLM→VLA→世界模型的关系，并指出无人机与地面机器人的 3 个核心差异。

### Step 2: 深度论文精读

不是泛读，而是对关键论文做**批判性分析**：

| 分析维度 | 要问的问题 |
|----------|-----------|
| **问题定义** | 论文要解决什么问题？这个问题重要吗？ |
| **假设** | 方法建立在哪些假设上？这些假设在无人机场景下成立吗？ |
| **实验设计** | 实验是否充分？消融实验是否完整？ |
| **局限性** | 作者承认了哪些局限？还有哪些没说的？ |
| **可复现性** | 代码是否开源？数据是否公开？结果能否复现？ |
| **与无人机的关联** | 该方法能直接用于无人机吗？需要哪些修改？ |

**具体做法**：
1. 选 3-5 篇核心论文（从 [论文导读合集](../06-论文导读合集/) 中选）
2. 对每篇论文填写上述分析表格
3. 找出论文之间的**共识和分歧**
4. 记录你的**疑问和想法**

### Step 3: 识别研究空白

这是从学习者到研究者的关键转折。研究空白通常出现在：

```
┌─────────────────────────────────────────────────┐
│              研究空白的四种来源                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  1. 方法空白：某类问题还没有好的解法              │
│     例：无人机实时VLA控制的延迟问题               │
│                                                 │
│  2. 场景空白：某方法在A领域成熟但B领域未探索      │
│     例：世界模型在游戏/自动驾驶成熟，无人机少见   │
│                                                 │
│  3. 评估空白：缺乏公平的对比基准                  │
│     例：无人机VLA没有统一的sim-to-real评估协议    │
│                                                 │
│  4. 融合空白：两个独立方向的交叉点                │
│     例：VLM场景理解 + 世界模型预测 → 无人机规划   │
│                                                 │
└─────────────────────────────────────────────────┘
```

**具体做法**：
1. 阅读每篇论文的 "Limitations" 和 "Future Work" 部分
2. 对比多篇论文，找共同提到但未解决的问题
3. 思考"无人机场景下，哪些问题更难？"
4. 参考 [研究空白与机会](02-研究空白与机会.md) 中的分析

### Step 4: 提出研究问题

好的研究问题需要满足 **FINER 标准**：

| 标准 | 含义 | 无人机领域示例 |
|------|------|---------------|
| **F**easible | 可行 | "用单GPU在仿真中训练无人机VLA" ✓ |
| **I**nteresting | 有趣 | "让无人机听懂自然语言指令" ✓ |
| **N**ovel | 新颖 | "用Transformer做无人机目标检测" ✗（太常规） |
| **E**thical | 合规 | "用无人机做人群监控" ✗（伦理敏感） |
| **R**elevant | 相关 | "提升无人机VLA的sim-to-real迁移" ✓ |

**研究问题模板**：
> 在 [场景] 下，如何解决 [问题]，使得 [指标] 提升？

示例：
- "在无人机FPV视角下，如何将世界模型的推理延迟降低到50Hz以下，使得实时避障成为可能？"
- "在缺乏真实飞行数据的情况下，如何利用视频生成世界模型合成训练数据，使得VLA策略的sim-to-real差距缩小30%？"

### Step 5: 设计实验验证

```
研究假设 → 实验设计 → 数据收集 → 模型训练 → 评估对比 → 结论
    ↑                                                    │
    └────────────── 迭代修正 ←──────────────────────────┘
```

**实验设计清单**：
- [ ] 基线选择：至少 2 个有代表性的基线方法
- [ ] 数据集：使用公开基准（如 UAVBench、AeroVerse）或说明自建数据集的必要性
- [ ] 评估指标：选择领域标准指标（成功率、推理延迟、泛化能力等）
- [ ] 消融实验：验证每个组件的贡献
- [ ] 统计显著性：多次运行取均值和标准差

## 推荐的研究切入点

基于当前领域状态，以下方向适合入门研究：

### 方向 A：VLM + 无人机场景理解（入门友好）
- **问题**：通用 VLM 在无人机俯视/斜视视角下表现差
- **方法**：在 GeoChat/UAVBench 上做 fine-tuning，对比通用 vs 领域适配效果
- **工具**：GeoChat 代码 + 无人机数据集
- **预期贡献**：一个轻量化的无人机视角适配方案

### 方向 B：世界模型 + 无人机仿真（中等难度）
- **问题**：无人机仿真器（AirSim/PX4 SITL）与真实环境差距大
- **方法**：用世界模型学习飞行动力学，替代部分物理仿真
- **工具**：gym-pybullet-drones + DreamerV3
- **预期贡献**：一个基于世界模型的无人机仿真增强方案

### 方向 C：VLA + 语言条件飞行控制（前沿方向）
- **问题**：现有 VLA 模型推理延迟高，无法满足无人机 50Hz 控制需求
- **方法**：知识蒸馏/量化，将大 VLA 压缩为轻量模型
- **工具**：OpenVLA/UAV-Flow + TensorRT
- **预期贡献**：一个实时化的无人机 VLA 部署方案
