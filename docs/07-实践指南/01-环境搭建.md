## 1. 硬件需求总览

不同项目对硬件的要求差异较大。以下表格列出各类型项目的最低与推荐配置：

### 硬件需求表

| 项目类型 | 典型代表 | GPU 显存（最低） | GPU 显存（推荐） | 内存（RAM） | 磁盘空间 | CPU |
|---|---|---|---|---|---|---|
| **VLM 推理（7B 参数）** | GeoChat, LLaVA | 16 GB | 24 GB (RTX 4090) | 32 GB | 100 GB SSD | 8 核+ |
| **VLM 微调（7B 参数）** | GeoChat Fine-tune | 24 GB | 48 GB (A6000) | 64 GB | 200 GB SSD | 16 核+ |
| **VLA 模型（OpenVLA 系列）** | UAV-Flow | 24 GB | 48 GB | 64 GB | 200 GB SSD | 8 核+ |
| **扩散模型训练** | FlightDiffusion | 24 GB | 80 GB (A100) | 64 GB | 500 GB SSD | 16 核+ |
| **扩散模型推理** | FlightDiffusion | 12 GB | 24 GB | 32 GB | 100 GB SSD | 8 核+ |
| **世界模型训练（DreamerV3）** | DreamerV3-Drone | 12 GB | 24 GB | 32 GB | 100 GB SSD | 8 核+ |
| **PX4 SITL + ROS2** | 仿真飞行测试 | 无需 GPU | — | 16 GB | 50 GB | 4 核+ |
| **数据集处理/可视化** | MotionScape | 无需 GPU | — | 16 GB | 500 GB+ | 4 核+ |

> **提示**：如果本地 GPU 不足，可考虑使用 Google Colab Pro（A100）、AutoDL 或 Lambda Cloud 等云 GPU 平台。

### 推荐硬件配置（个人开发）

- **GPU**：NVIDIA RTX 4090 (24 GB) 或 RTX 3090 (24 GB)
- **CPU**：AMD Ryzen 9 / Intel i9，16 核以上
- **内存**：64 GB DDR4/DDR5
- **存储**：1 TB NVMe SSD（系统+环境）+ 2 TB HDD（数据集）
- **操作系统**：Windows 11 + WSL2 (Ubuntu 22.04) 或原生 Ubuntu 22.04

## 2. NVIDIA 驱动与 CUDA 安装

### 2.1 确认 GPU 型号与当前驱动

```bash
# 在 Windows PowerShell 中查看 GPU 信息
nvidia-smi

# 输出示例：
# NVIDIA-SMI 546.xx   Driver Version: 546.xx   CUDA Version: 12.x
# GPU Name        | Bus-Id
# NVIDIA GeForce RTX 4090 | 00000000:01:00.0
```

### 2.2 安装/更新 NVIDIA 驱动

访问 [NVIDIA 官方驱动下载页面](https://www.nvidia.com/Download/index.aspx)，选择对应 GPU 型号下载最新 Game Ready Driver 或 Studio Driver。

### 2.3 CUDA Toolkit 安装

当前 PyTorch 主流版本支持 CUDA 11.8 和 CUDA 12.1。建议安装 CUDA 12.1。

**Windows 安装步骤：**

1. 访问 [CUDA Toolkit Archive](https://developer.nvidia.com/cuda-toolkit-archive)
2. 下载 CUDA Toolkit 12.1
3. 运行安装程序，选择"自定义安装"，勾选 CUDA Runtime、CUDA Developer Tools
4. 安装完成后验证：

```bash
nvcc --version
# 应显示：Cuda compilation tools, release 12.1
```

**WSL2 中的 CUDA：**

WSL2 中无需单独安装 CUDA Toolkit，Windows 端的驱动会通过 `/usr/lib/wsl/lib/` 自动映射到 WSL2 内部。只需确保 Windows 端驱动已正确安装。

```bash
# 在 WSL2 终端中验证
nvidia-smi
# 应能看到 GPU 信息
```

### 2.4 cuDNN 安装（可选，但推荐）

cuDNN 通常已包含在 PyTorch 的预编译包中，无需单独安装。如果需要从源码编译 PyTorch 或使用 TensorFlow，请从 [NVIDIA cuDNN](https://developer.nvidia.com/cudnn) 下载安装。

## 4. ROS2 Humble 在 WSL2 中的安装

> 适用于需要 ROS2 + PX4 联合仿真的项目（如 UAV-Flow 的仿真评估环境）。

### 4.1 系统准备

```bash
# 确保 WSL2 使用 Ubuntu 22.04
wsl --list --verbose
# 如果不是 22.04，安装：
wsl --install -d Ubuntu-22.04
```

### 4.2 安装 ROS2 Humble

```bash
# 设置语言环境
sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# 添加 ROS2 apt 源
sudo apt install software-properties-common
sudo add-apt-repository universe
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# 安装 ROS2 Humble
sudo apt update
sudo apt upgrade
sudo apt install ros-humble-desktop

# 安装开发工具
sudo apt install ros-dev-tools

# 配置环境
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

### 4.3 验证 ROS2 安装

```bash
# 终端 1：启动 talker
ros2 run demo_nodes_cpp talker

# 终端 2：启动 listener
ros2 run demo_nodes_py listener
# 应能看到消息收发
```

### 4.4 安装 colcon 构建工具

```bash
sudo apt install python3-colcon-common-extensions
```

## 7. 环境验证清单

完成上述安装后，运行以下脚本验证环境完整性：

```python
"""
环境验证脚本 - 检查 UAV VLA/VLM/World Model 项目所需组件
运行方式: python check_env.py
"""

def check_package(name, import_name=None):
    """检查 Python 包是否安装"""
    import_name = import_name or name
    try:
        mod = __import__(import_name)
        version = getattr(mod, '__version__', 'unknown')
        print(f"  [OK] {name}: {version}")
        return True
    except ImportError:
        print(f"  [MISSING] {name}")
        return False

def check_all():
    print("=" * 50)
    print("UAV VLA/VLM/World Model 环境检查")
    print("=" * 50)

    # 1. CUDA
    import torch
    print(f"\n[GPU]")
    print(f"  CUDA available: {torch.cuda.is_available()}")
    if torch.cuda.is_available():
        print(f"  GPU: {torch.cuda.get_device_name(0)}")
        print(f"  VRAM: {torch.cuda.get_device_properties(0).total_mem / 1e9:.1f} GB")

    # 2. 核心包
    print(f"\n[Core]")
    for pkg in ['torch', 'torchvision', 'torchaudio']:
        check_package(pkg)

    # 3. Hugging Face
    print(f"\n[Hugging Face]")
    for pkg in ['transformers', 'diffusers', 'datasets', 'accelerate', 'peft', 'huggingface_hub']:
        check_package(pkg)

    # 4. 科学计算
    print(f"\n[Scientific]")
    for pkg in ['numpy', 'scipy', 'matplotlib', 'opencv-python', 'PIL:pillow']:
        if ':' in pkg:
            name, imp = pkg.split(':')
            check_package(name, imp)
        else:
            check_package(pkg)

    # 5. 机器人与仿真
    print(f"\n[Robotics & Sim]")
    for pkg in ['gymnasium', 'pyquaternion', 'transforms3d']:
        check_package(pkg)

    print(f"\n{'=' * 50}")
    print("环境检查完成。如果所有项均为 [OK]，可以开始实验。")

if __name__ == '__main__':
    check_all()
```

## 8. 常见问题与解决方案

### Q1: `torch.cuda.is_available()` 返回 `False`

**原因与解决方案：**

- 检查 NVIDIA 驱动是否安装正确：运行 `nvidia-smi`
- PyTorch 版本与 CUDA 版本不匹配：重新安装对应版本的 PyTorch
- WSL2 中未安装 `nvidia-cuda-toolkit`：确保 Windows 端驱动是最新版本
- 多 GPU 环境中 CUDA_VISIBLE_DEVICES 设置错误：检查环境变量

### Q2: WSL2 中内存不足（OOM）

```bash
# 在 Windows 端创建 %UserProfile%\.wslconfig 文件
# 内容如下：
[wsl2]
memory=48GB
swap=8GB
processors=16
```

### Q3: Hugging Face 模型下载速度慢

```bash
# 使用镜像源
export HF_ENDPOINT=https://hf-mirror.com

# 或使用 huggingface-cli 下载
huggingface-cli download --resume-download model_name --local-dir ./models
```

### Q4: ROS2 与 Python 版本冲突

ROS2 Humble 默认使用 Python 3.10。建议在 ROS2 工作空间使用系统 Python，在深度学习环境使用 conda/venv。两者不要混用。

```bash
# 在 ROS2 终端中
source /opt/ros/humble/setup.bash
# 不要在此环境下 conda activate

# 在深度学习终端中
conda activate uavvla
# 不要在此环境下 source ROS2 的 setup.bash
```

### Q5: PX4 SITL 编译失败

```bash
# 确保安装了所有依赖
sudo apt install -y git cmake ninja-build gperf \
  ccache dfu-util device-portal-common \
  libglib2.0-dev libpython3-dev pkg-config

# 清理后重新编译
cd PX4-Autopilot
make clean
make px4_sitl_default
```

### Q6: `xformers` 安装失败

```bash
# 确保 PyTorch 和 CUDA 版本匹配
pip install xformers --index-url https://download.pytorch.org/whl/cu121

# 或使用预编译版本
pip install xformers==0.0.23.post1 --extra-index-url https://download.pytorch.org/whl/cu121
```

### Q7: GPU 显存不足怎么办？

```bash
# 方法 1: 使用半精度推理
model = model.to(torch.float16)

# 方法 2: 使用 4-bit 量化（需要 bitsandbytes）
from transformers import BitsAndBytesConfig
bnb_config = BitsAndBytesConfig(load_in_4bit=True)
model = AutoModelForCausalLM.from_pretrained(model_name, quantization_config=bnb_config)

# 方法 3: 使用 DeepSpeed ZeRO
# 方法 4: 减小 batch_size
# 方法 5: 使用 gradient checkpointing
model.gradient_checkpointing_enable()
```
