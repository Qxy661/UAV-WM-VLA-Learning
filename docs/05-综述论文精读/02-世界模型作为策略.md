# 世界模型作为策略 (World Model as Policy)

> 预计阅读：15 分钟 | 前置知识：了解世界模型和VLA基础（见Part 1）

---

## 核心思想

将世界模型直接用作或辅助机器人策略，而不是分离的组件。

传统方法：感知模块 → 规划模块 → 控制模块
世界模型策略：一个统一的模型，同时理解世界并输出动作

---

## 五大范式

综述将世界模型策略分为五种范式：

```
┌──────────────────────────────────────────────────────────┐
│              World Model as Policy: 五大范式              │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  1. IDM风格策略 (Inverse Dynamics Model)                 │
│     先预测未来视觉 → 再用IDM提取动作                       │
│                                                          │
│  2. 单骨干策略 (Single-backbone)                          │
│     一个模型同时建模视频和动作                              │
│                                                          │
│  3. MoE/MoT风格策略                                      │
│     视频和动作专家分离，通过注意力交互                      │
│                                                          │
│  4. 统一VLA模型 (Unified VLA)                             │
│     世界建模作为训练目标内化到VLA中                        │
│                                                          │
│  5. 隐空间世界建模 (Latent-space)                         │
│     在表示空间做未来预测，不生成像素                        │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

---

## 范式一：IDM风格策略

### 工作流程

```
输入: 当前图像 + 语言指令
         ↓
   生成未来视觉轨迹 (视频预测)
         ↓
   逆动力学模型 (IDM) 从轨迹中恢复动作
         ↓
输出: 动作序列
```

### 关键变体

**早期子目标图像方法**：
- **SuSIE**：用图像编辑扩散模型预测子目标图像
- **GHIL-Glue**：用扩散模型生成中间目标

**视频-IDM方法**：
- **UniPi**：统一视频生成+IDM的框架
- **GR-1** (ByteDance)：视频生成+动作预测统一
- **VPP**：视频预测策略

**3D感知IDM扩展**：
- **Hind4sight-Net**：从视频中提取3D感知运动结构
- **AVDC**：3D感知的视频扩散控制

### 优势与劣势

| 优势 | 劣势 |
|------|------|
| 模块化，可解释 | 需要单独训练IDM |
| 视频预测可复用 | 视频生成质量影响动作质量 |
| 适合长距离规划 | 推理延迟较高 |

---

## 范式二：单骨干策略

### 工作流程

```
输入: 当前图像 + 语言指令
         ↓
   统一骨干网络 (同时处理视频和动作)
         ↓
输出: 动作 + 视频预测 (联合输出)
```

### 代表模型

- **UVA**：统一视频-动作模型
- **UWM**：统一世界模型
- **VideoVLA**：视频VLA
- **Cosmos Policy**：基于Cosmos的策略
- **DreamZero** (WAM)：零样本世界模型策略

### 优势与劣势

| 优势 | 劣势 |
|------|------|
| 端到端训练，简洁 | 模型规模大 |
| 视频和动作共享表示 | 训练数据需求高 |
| 推理效率高 | 可解释性差 |

---

## 范式三：MoE/MoT风格策略

### 工作流程

```
输入: 当前图像 + 语言指令
         ↓
   ┌─────────────┐  ┌─────────────┐
   │ 视频专家    │  │ 动作专家    │
   │ (Video Expert)│ │(Action Expert)│
   └──────┬──────┘  └──────┬──────┘
          │                │
          └─── 交叉注意力 ───┘
                    ↓
              融合输出
```

### 代表模型

- **GE-Act**：专家耦合设计
- **Motus**：混合Transformer
- **BagelVLA**：VLA的MoT设计
- **WoG (World Guidance)**：世界模型指导动作生成

### 优势与劣势

| 优势 | 劣势 |
|------|------|
| 专业化分工 | 架构复杂 |
| 可独立优化 | 交互机制设计困难 |
| 灵活性高 | 训练不稳定 |

---

## 范式四：统一VLA模型

### 核心思想

将世界建模作为训练目标内化到单一VLA骨干中：

```
训练目标 = 动作预测损失 + 视频预测损失 + 语言理解损失
```

### 代表模型

- **GR-1/GR-2** (ByteDance)：视频生成+动作预测统一
- **UP-VLA**：统一策略VLA
- **DreamVLA**：带世界建模的VLA
- **UniVLA**：任务中心隐动作
- **InternVLA-A1**：大规模统一VLA

### 优势与劣势

| 优势 | 劣势 |
|------|------|
| 最简洁的架构 | 训练目标复杂 |
| 多任务学习效果好 | 需要大量数据 |
| 统一的表示空间 | 单一模型的风险 |

---

## 范式五：隐空间世界建模

### 核心思想

在表示空间中做未来预测，不显式生成像素：

```
输入: 当前状态表示 + 动作
         ↓
   JEPA风格预测器 (在隐空间预测未来表示)
         ↓
输出: 下一状态表示 (非像素)
```

### 代表模型

- **FLARE**：隐空间未来预测
- **VLA-JEPA**：JEPA用于VLA
- **VISTA**：视觉隐空间时间预测
- **JEPA-VLA**：JEPA风格的VLA
- **DIAL**：动态隐空间动作学习

### 优势与劣势

| 优势 | 劣势 |
|------|------|
| 计算效率高 | 无法生成可视化预测 |
| 避免像素空间的高成本 | 表示空间的设计关键 |
| 更适合实时应用 | 可解释性差 |

---

## 五种范式对比

| 维度 | IDM | 单骨干 | MoE/MoT | 统一VLA | 隐空间 |
|------|-----|--------|---------|---------|--------|
| **架构复杂度** | 中 | 低 | 高 | 低 | 中 |
| **训练难度** | 中 | 高 | 高 | 高 | 中 |
| **推理效率** | 低 | 高 | 中 | 高 | 高 |
| **可解释性** | 高 | 低 | 中 | 低 | 低 |
| **视频生成** | ✓ | ✓ | ✓ | 部分 | ✗ |
| **实时性** | 差 | 好 | 中 | 好 | 好 |
| **适合无人机** | 中 | 好 | 中 | 好 | 好 |

| IDM | ANWM (FFP + IDM) |
