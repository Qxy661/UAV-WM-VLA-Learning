# 什么是视觉语言动作模型 (VLA)

> 预计阅读：15 分钟 | 前置知识：了解VLM（见[什么是VLM](02-什么是VLM.md)）

---

## 一句话定义

**VLA（Vision-Language-Action Model）是能同时理解视觉、语言并输出动作的端到端模型。**

简单来说：VLA = VLM + 动作输出。它不仅能"看图说话"，还能"看图做事"。

---

## 直觉理解

想象你操控一架无人机，看到前方有障碍物：

- **VLM**："前方有一棵树，建议绕行"（只能说，不能做）
- **VLA**：看到树 → 直接输出动作指令"向左偏移2米"（看+说+做一体化）

VLA 的核心突破是**端到端**：从感知到决策到动作，一个模型搞定，不需要中间模块。

---

## 从 VLM 到 VLA 的演进

```
传统机器人系统:
  感知模块 → 规划模块 → 控制模块 → 执行
  (每个模块独立设计和训练)

VLM 驱动的系统:
  VLM(感知+理解) → 规划模块 → 控制模块 → 执行
  (感知和理解统一，但规划和控制仍分开)

VLA 端到端系统:
  VLA(感知+理解+决策+动作) → 执行
  (一个模型端到端)
```

---

## VLA 的核心架构

```
┌───────────┐   ┌───────────┐   ┌───────────┐
│  视觉输入  │ + │  语言指令  │ + │ 历史状态  │
└─────┬─────┘   └─────┬─────┘   └─────┬─────┘
      │               │               │
      └───────────────┼───────────────┘
                      ↓
              ┌───────────────┐
              │   VLA 模型    │
              │ (多模态骨干)  │
              └───────┬───────┘
                      ↓
              ┌───────────────┐
              │  动作输出头   │
              │ (Action Head) │
              └───────┬───────┘
                      ↓
              ┌───────────────┐
              │   执行动作    │
              │ (关节/速度)   │
              └───────────────┘
```

### 动作表示

VLA 的动作输出有多种方式：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **离散动作** | 预定义动作集合（前进/左转/悬停） | 简单任务 |
| **连续动作** | 直接输出速度/加速度值 | 精细控制 |
| **航点序列** | 输出未来N个目标位置 | 路径规划 |
| **token化动作** | 将动作编码为token，和文本统一处理 | 统一架构 |

## VLA 的关键创新

### 1. 动作 Token 化

将动作编码为离散token，与文本token在同一个序列中处理：

```
输入: <图像tokens> "飞向红色建筑" <历史动作tokens>
输出: "好的，我将" <动作token_1> <动作token_2> ...
```

**代表**：RT-2 (Google, 2023) — 首次将动作token化用于机器人控制

### 2. 流匹配 (Flow Matching)

用连续归一化流生成动作分布，而非离散采样：

```
噪声 → 流匹配网络 → 连续动作分布 → 采样 → 精细动作
```

**代表**：π₀ (Physical Intelligence, 2024)

### 3. 扩散策略 (Diffusion Policy)

用扩散模型生成动作序列：

```
随机噪声 → 去噪N步 → 动作序列
```

**代表**：Diffusion Policy (Chi et al., 2023)

## VLA vs VLM vs 传统控制

| 维度 | 传统控制 | VLM + 控制器 | VLA 端到端 |
|------|---------|-------------|-----------|
| **感知** | 手工设计特征 | VLM自动提取 | 模型内置 |
| **决策** | 规则/优化 | LLM规划 | 端到端 |
| **动作** | 控制器输出 | 控制器输出 | 模型直接输出 |
| **训练** | 模块独立训练 | 模块独立训练 | 端到端训练 |
| **泛化** | 差 | 好 | 好 |
| **可解释性** | 好 | 中等 | 差 |
| **安全性** | 可验证 | 可验证 | 难验证 |

## 无人机 VLA 的特殊挑战

### 1. 动作空间复杂

地面机器人：2D动作 (x, y, θ)
无人机：4D动作 (roll, pitch, yaw, thrust) + 6-DoF动力学

```
地面: a_t = [线速度, 转向角]  →  2维
无人机: a_t = [滚转, 俯仰, 偏航, 油门]  →  4维 (甚至更多)
```

### 2. 安全约束严格

- 地面机器人碰撞：可能只是撞墙停下
- 无人机碰撞：坠机、损坏、伤人

**解决方案**：VLA-AN 引入"几何安全校正"，替换随机生成策略

### 3. 实时性要求高

无人机飞行动力学快，需要高频控制：
- 地面机器人：10-50 Hz 控制频率
- 无人机：50-200 Hz 控制频率

**解决方案**：VLA-AN 实现 8.3x 推理加速

### 4. Sim-to-Real 差距大

无人机3D动力学仿真比地面2D运动仿真更难：
- 空气动力学效应难以精确建模
- 3D环境比2D环境更复杂

**解决方案**：用3D Gaussian Splatting生成高保真训练数据

## 无人机 VLA 的典型架构

以 VLA-AN 为例的三阶段训练：

```
阶段1: 场景理解
  输入: 航拍图像 + 语言描述
  目标: 理解场景语义
  训练: 视觉问答、图像描述

阶段2: 飞行技能
  输入: 当前状态 + 简单指令
  目标: 学习基本飞行动作
  训练: 模仿学习、简单导航

阶段3: 复杂导航
  输入: 当前状态 + 复杂指令
  目标: 复杂环境中的自主导航
  训练: 强化学习、多任务学习
```

## VLA 的未来趋势

1. **更大规模**：从7B到70B+参数，更强的泛化能力
2. **更多模态**：加入深度、点云、音频等
3. **更强安全**：形式化安全验证
4. **更低延迟**：模型压缩、量化、硬件加速
5. **跨形态**：一个VLA控制多种机器人（地面+空中）

## 思考题

1. **基本理解**：VLA和VLM的核心区别是什么？为什么需要VLA而不能只用VLM？

2. **架构思考**：动作token化有什么优势和劣势？

3. **无人机应用**：为无人机设计VLA时，动作空间应该怎么定义？

4. **安全分析**：VLA端到端方案的"黑箱"特性在无人机场景下有什么风险？如何缓解？

5. **进阶思考**：VLA能否同时输出"解释"和"动作"？即不仅做事，还能解释为什么这么做？

<details>
<summary>参考答案</summary>

1. VLM只输出文本理解，VLA还输出动作指令。需要VLA是因为端到端系统更简洁、泛化更好，避免模块间信息损失。

2. 优势：统一架构，可以利用语言模型的强大能力。劣势：动作精度可能受限于token化粒度。

3. 4D连续动作 (roll, pitch, yaw, thrust) 是最常见的；也可以用航点序列表示；或者离散动作集合。

4. 黑箱特性意味着难以验证安全性。缓解：加入几何安全校正模块、物理约束、置信度阈值、多传感器融合验证。

5. 可以，通过在输出中同时包含文本解释和动作token。这有助于可解释性和调试。
</details>
