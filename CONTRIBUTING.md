# 贡献指南

感谢你对本项目的关注！欢迎通过以下方式贡献。

---

## 如何贡献

### 1. 报告问题

发现文档错误、链接失效、内容过时？请提交 [Issue](https://github.com/Qxy661/UAV-WM-VLA-Learning/issues)：

- 描述问题所在（文件名+行号）
- 说明正确的内容应该是什么
- 如果有相关论文/资料，请附上链接

### 2. 补充内容

想补充新的论文解读、实践指南或学习笔记？

1. Fork 本仓库
2. 在对应目录下创建新文件
3. 遵循现有文档格式（见下方模板）
4. 提交 Pull Request

### 3. 改进现有文档

发现表述不清、结构混乱、缺少图表？

1. Fork 本仓库
2. 修改对应文件
3. 提交 Pull Request，说明修改原因

---

## 文档格式模板

每篇文档应遵循以下格式：

```markdown
# 标题

> 预计阅读：X 分钟 | 前置知识：xxx

---

## 核心内容

（配图、配表、配公式简化版）

## 关键论文

（引用相关论文，含 arXiv 链接）

## 延伸阅读

（相对路径链接到其他文档）

## 思考题

1. 问题一
2. 问题二

<details>
<summary>参考答案</summary>

答案内容
</details>
```

## 内容规范

- **语言**：中文为主，技术术语保留英文
- **风格**：结构化、表格优先、有优先级标注
- **链接**：使用相对路径，确保 GitHub 上可点击
- **图表**：优先使用 Mermaid（GitHub 原生支持）
- **论文引用**：必须包含 arXiv 链接或 DOI

---

## 补充新论文的标准

如果你想补充一篇新的论文解读，请确保：

1. **与无人机领域相关**：纯地面机器人论文不在范围内
2. **有足够的创新性**：不是简单的增量工作
3. **包含以下信息**：
   - 一句话总结
   - 核心贡献（3-5 点）
   - 方法要点
   - 实验结果亮点
   - 论文链接和代码链接（如有）

## 行为准则

- 尊重所有贡献者
- 保持专业和友善的讨论
- 不发布侵权内容
- 不发布与项目无关的广告
